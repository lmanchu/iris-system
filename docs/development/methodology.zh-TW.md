# 🛠️ Iris 開發方法論

> Iris 如何進行軟體開發專案

[English](./methodology.md) | 繁體中文

## 核心原則

### 1. 記憶驅動開發
- 在每次對話開始時透過 `/iris` slash command 載入上下文
- 跨對話維護長期記憶
- 參考過去的決策和學習

### 2. 迭代式問題解決
- 將複雜問題分解為可管理的步驟
- 增量測試
- 記錄解決方案供未來參考

### 3. 全面的文檔
- 邊開發邊寫文檔
- 解釋決策和權衡
- 為使用者和開發者創建指南

## 開發工作流程

### 階段 1：研究與規劃
1. **理解需求**
   - 釐清使用者需求
   - 研究現有解決方案
   - 定義範圍

2. **技術設計**
   - 選擇適當的架構
   - 選擇技術和工具
   - 規劃文件結構

3. **文檔化計劃**
   - 創建 TODO 列表
   - 概述實現步驟
   - 設定成功標準

### 階段 2：實現
1. **建立核心功能**
   - 從 MVP（最小可行產品）開始
   - 測試每個組件
   - 根據反饋迭代

2. **問題解決**
   - 系統性除錯
   - 記錄問題和解決方案
   - 從錯誤中學習

3. **整合**
   - 連接組件
   - 端到端測試
   - 性能優化

### 階段 3：文檔與發布
1. **使用者文檔**
   - 安裝指南
   - 使用說明
   - 疑難排解技巧

2. **開發者文檔**
   - 代碼架構
   - API 參考
   - 貢獻指南

3. **版本控制**
   - Git commits 清晰訊息
   - 語義化版本
   - Changelog 維護

4. **發布流程**
   - 創建 GitHub releases
   - 添加 topics/tags
   - 規劃未來開發

## 案例研究：Iris Immersive Translate

### 時程：2025-11-01（6 小時）

**階段 1：研究（1 小時）**
- 研究沉浸式翻譯概念
- 探索 Ollama 本地 LLM 整合
- 決定採用 Chrome Extension 方式

**階段 2：實現（4 小時）**
- 建立 Manifest V3 Chrome Extension
- 整合 Ollama API
- 用 LaunchAgent 解決 CORS 問題
- 用 CSS 創建雙語 UI

**階段 3：文檔（1 小時）**
- 撰寫全面的 README
- 創建安裝指南
- 設置版本管理
- 發布到 GitHub

### 關鍵學習
1. **CORS 配置** - Chrome extensions 需要明確的 OLLAMA_ORIGINS 設置
2. **快捷鍵衝突** - 檢查平台衝突（BrowserOS）
3. **模型彈性** - 支援所有 Ollama 模型，而非只有一個
4. **文檔優先** - 好的文檔 = 更容易採用

## 工具與技術

### 開發工具
- **Claude Code** - 主要開發環境
- **Git** - 版本控制
- **GitHub** - 代碼託管和協作

### 整合工具
- **MCP (Model Context Protocol)** - API 整合
- **BrowserOS** - 瀏覽器自動化測試
- **Node.js** - 腳本和自動化

### 文檔工具
- **Markdown** - 所有文檔
- **GitHub Pages** -（未來）文檔託管

- 撰寫清晰、有註釋的代碼
- 遵循一致的風格
- 測試邊緣情況

### 使用者體驗
- 清晰的錯誤訊息
- 直覺的界面
- 全面的幫助文檔

### 維護
- 版本管理（CHANGELOG.md）
- 問題追蹤
- 定期更新
